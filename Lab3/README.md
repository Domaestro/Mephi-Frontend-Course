# Пояснения по переделанным лабораторным

Ниже коротко объяснено, почему код написан именно так.

## HTML Basics, DOM API

### Задание 1
- CSS не использовался вообще, потому что вы отдельно указали, что для этого PDF нужно сделать все средствами HTML.
- `Добро пожаловать!` сделано через `h1`, потому что в образце оно выглядит как обычный крупный заголовок браузера.
- `Приветствуем` сделано через `i`, потому что в образце слово выделено курсивом.
- `вас` сделано через `u`, потому что в образце слово подчеркнуто.
- `в нашем` сделано через `b`, потому что в образце это жирный текст.
- `магазине` сделано ссылкой `a`, потому что в браузере ссылка по умолчанию синяя и подчеркнутая, а в образце именно такой вид.
- Строка `&lt;h1&gt;Теорема Пифагора&lt;/h1&gt;` выведена как текст, а не как HTML-тег, потому что в условии это нужно показать именно на экране, а не превратить в настоящий заголовок.
- Степени оформлены через `sup`, потому что это стандартный HTML-способ показать верхний индекс без CSS.
- Таблица сделана именно тегом `table`, а границы заданы через HTML-атрибуты `border`, `cellspacing`, `cellpadding`, потому что в этом задании нельзя уходить в CSS.
- Перечеркнутая старая цена сделана через `s`, потому что в образце число 1000 перечеркнуто.

### Задание 2
- Основа страницы оставлена той же, что и в первом задании, потому что по условию нужно именно защитить предыдущую страницу, а не сделать новую.
- `selectstart` запрещает запуск выделения мышью.
- `copy` запрещает событие копирования.
- `contextmenu` запрещает меню по правой кнопке мыши.
- `keydown` с проверкой `Ctrl/Cmd + C` и `Ctrl/Cmd + A` добавлен не для украшения, а чтобы защита была не только мышью, но и с клавиатуры.
- Здесь использован именно DOM API: обработчики навешаны через `document.addEventListener(...)`.

### Задание 3
- На странице оставлены только кнопка и фраза `Счетчик: ...`, потому что вы отдельно это потребовали.
- Значение хранится в `localStorage`, потому что по условию счетчик не должен сбрасываться после перезагрузки браузера.
- Отдельная функция `render()` нужна, чтобы обновление текста было в одном месте, а не дублировалось.

### Задание 4
- Для времени взят `Date.now()`, потому что это прямой и стандартный способ получить текущее время в миллисекундах в браузере.
- В `localStorage` хранится не только счетчик, но и время первого клика, потому что без этого среднее количество кликов в секунду после перезагрузки посчитать нельзя.
- Среднее считается по формуле: `количество кликов / прошедшее время в секундах`.
- Значение выводится через `toFixed(2)`, чтобы число не было слишком длинным и выглядело читаемо.

### Задание 5
- Здесь парсинга выражения нет специально, потому что в условии прямо сказано: калькулятор должен поддерживать одно действие за раз.
- Поэтому код хранит только три вещи: первое число, оператор и второе число.
- После нажатия оператора калькулятор ждет второе число, а после `=` выполняет ровно одну операцию.
- Деление на ноль отдельно обрабатывается как ошибка, потому что это отдельный случай, который нельзя молча пропускать.

### Задание 6
- Здесь уже нужен полный ввод выражения со скобками, поэтому простой подход из пятого задания уже не подходит.
- Использована обратная польская нотация, потому что именно этого требует условие.
- Сначала строка разбивается на токены: числа, операторы и скобки.
- Потом выражение переводится в ОПН алгоритмом сортировочной станции: он учитывает приоритет `*` и `/` выше, чем `+` и `-`, и правильно обрабатывает скобки.
- После этого ОПН вычисляется стеком. Это стандартный способ для таких заданий.
- Поддержан унарный минус, чтобы выражения вроде `-2+3` или `-(2+3)` тоже считались корректно.

## CSS

### Задание 1
- Здесь CSS уже нужен, потому что в PDF образец визуально отличается от обычного HTML.
- Для разноцветных букв текст разбит на `span`, потому что по-другому покрасить отдельные буквы нельзя.
- Для курсива использован `font-style: italic`.
- Для жирного текста использован `font-weight: 700`.
- Для подчеркивания использован `text-decoration: underline`.
- Цвет таблицы задан через `background`, потому что по образцу она должна быть бирюзовой.
- Границы таблицы заданы через `border` и `border-collapse`, чтобы линии выглядели как единый контур.
- Для строки с `Теорема Пифагора` выбран отдельный шрифт, потому что в образце эта часть отличается от остального текста.

### Задание 2
- Использованы `@media`, потому что по условию надпись должна зависеть от ширины окна, а JS использовать нельзя.
- Границы взяты ровно из условия: 1024 px и 768 px.
- Три отдельных блока `Desktop`, `Tablet`, `Mobile` проще и надежнее, чем пытаться менять текст у одного элемента без JavaScript.

### Задание 3
- Меню сделано через `position: fixed`, потому что так оно гарантированно остается на месте при прокрутке страницы.
- У `body` добавлен верхний отступ, чтобы содержимое не ушло под фиксированное меню.
- Дополнительный текст внизу нужен только для появления прокрутки и проверки поведения меню.

### Задание 4
- Круг делается из обычного `div` через `width: 60px`, `height: 60px` и `border-radius: 30px`.
- Именно `30px`, а не `50%`, потому что в условии радиус задан явно числом 30 px.
- Синий цвет задан через `background: blue`.

### Задание 5
- Круг оставлен таким же, как в четвертом задании, чтобы сохранялась логика условия: сначала сделать круг, потом заставить его двигаться.
- Использовано `position: absolute`, потому что тогда положение элемента можно напрямую менять через `left` и `top`.
- Координаты берутся из события `mousemove`, потому что именно оно сообщает текущее положение курсора.
- Из координат вычитается 30, чтобы курсор был в центре круга, а не в его левом верхнем углу.

### Задание 6
- Страница сверстана блоками, потому что на образце видны отдельные прямоугольные области: меню, центральный блок, три колонки и футер.
- Внешний контейнер фиксированной ширины нужен, чтобы макет был похож на изображение из PDF, где весь контент расположен по центру.
- Три колонки сделаны через `flex`, потому что это самый прямой способ поставить три одинаковых блока в один ряд.
- Цвет ссылок и фона блоков подобран под образец из PDF, а не произвольно.
